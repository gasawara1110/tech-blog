# なぜ技術発信を始めるのか

## HW屋視点のロボット関連の技術共有

私が学生時代にロボットについて勉強し始めた頃は、Makerムーブメントが盛り上がっていた時期でした。  
Arduino や Raspberry Pi、3Dプリンタなどが広がり、個人でもロボットや電子工作に挑戦しやすくなっていく空気がありました。

そこから十数年が経ち、ロボットを取り巻く環境はかなり変わったと感じています。  
AI、センサ、モータ、エッジデバイス、シミュレーション環境、製造技術など、関連する技術の進化がとても速いです。

正直、海外のロボット・ハードウェア企業のスピード感には驚かされることが多いです。  
完成度の高いモータユニット、扱いやすい開発キット、実機デモをどんどん公開していく姿勢を見ると、刺激を受けると同時に、自分も手を動かして学び続けないといけないなと感じます。

ロボットは、ソフトウェアだけでもハードウェアだけでも成立しません。  
メカ、回路、組み込み、制御、電源、通信、センサ、ROS 2 などがつながって、ようやく一つのシステムとして動きます。

私はどちらかというとハードウェア寄りのエンジニアですが、ロボット開発ではこの「境界領域」こそが面白いと思っています。  
うまく動かない原因が、制御なのか、通信なのか、電源なのか、メカなのか分からない。少なくとも今のところ、AIだけで全部きれいに切り分けられる領域ではないと思っています。  
でも、そこを一つずつ切り分けて、最後にちゃんと動いた瞬間が一番楽しい。

このブログでは、そういう HW屋視点から、ロボットを動かすために必要な技術を整理していきたいと思っています。  
部品紹介や理論解説だけではなく、「なぜその構成にするのか」「どこで詰まるのか」「実際に動かすと何が難しいのか」まで含めて書いていきたいです。

## 作りたいものを作って言語化する

ロボット技術は、読んで理解するだけではなかなか身につきません。  
シミュレーションを作る、制御器を実装する、動かして失敗する、ログを見る、仮説を直す。  
その繰り返しで少しずつ理解が深まるものだと思っています。

このブログでは、作ったものを単に「できました」で終わらせず、なぜそうしたのか、どこで詰まったのか、何が分かったのかを言語化していきます。

## 技術的な試行錯誤を後から見返せる形で残す

自分が何を考え、何を作り、どのように検証してきたかを後から見返せる場所を作りたいと思っています。

会社で作ったものをそのまま公開することはできませんし、対外的にも職務経歴書だけでは日々の学習や小さな実験、設計時に考えたことまではなかなか伝わりません。  
だからこそ、自分の名前で公開できる範囲の技術メモや実験記録を積み上げていきたいと考えています。

# おわりに

まずは、MuJoCo を使ったシミュレーションや、ロボット制御、モータ・電源まわりの技術メモから少しずつ書いていく予定です。

最初から完成度の高い記事ばかりを書くのは難しいと思いますが、作ったもの、試したこと、失敗したこと、そこから分かったことを、できるだけ正直に残していきたいです。

ロボットは、実際に動かしてみると想定外のことばかり起きます。  
だからこそ、理論だけでなく、実装や検証の過程も含めて言語化していく価値があると思っています。

このブログが、ロボットを作りたい人、制御やハードウェアに興味がある人、そして未来の自分にとって、少しでも役に立つ記録になれば嬉しいです。
